# ZhuQian 文件标准与功能设计 v2.0

## 一、设计理念

竹签 = Markdown + 用户自定义语义标签 + 可选模板引导

核心原则：

1. **标签由用户定义** — 系统只提供语法机制，不预设标签类型
2. **标签定界符可自定义** — 默认 `[]`，用户可换为 `《》`、`{}` 等
3. **标签可隐藏** — 一键切换，隐藏后得到纯净内容
4. **渐进增强** — 纯文本 → Markdown → 语义标签 → 模板引导
5. **工具无关** — .zq 文件在任何文本编辑器中可读

## 二、文件格式

### 2.1 文件结构

```
---zq-meta---
{
  "template": "freeform",
  "font_size": 16.0,
  "font_name": "times",
  "markdown_render": true,
  "hide_labels": false,
  "language": "Zh",
  "label_delimiter": ["[", "]"],
  "theme": { "name": "默认", "rules": [...] }
}
---end-meta---

（正文：Markdown + 语义标签）
```

- 元数据区：`---zq-meta---` 到 `---end-meta---`，JSON 格式
- 正文区：其余全部为文档内容
- 扩展名：`.zq`（兼容 `.txt` / `.md`）

### 2.2 正文语法

正文 = 标准 Markdown + 语义标签扩展

Markdown 全部保留：
```
# 标题  **粗体**  *斜体*  ~~删除线~~  `代码`
- 列表  > 引用  [链接](url)
```

### 2.3 语义标签语法

定界符由 `label_delimiter` 配置决定，默认为 `[]`。以下以默认定界符为例。

#### A) 标注标签 — 给内容加语义类型

```
张三[角色]推开门。
北京[地点]下着大雨。
```

| 模式 | 显示 |
|------|------|
| 显示标签 | 张三**[角色]**推开门。（标签高亮） |
| 隐藏标签 | 张三推开门。 |

`[]`前的文字是被标注的内容，`[]`内是用户自定义的类型名。

#### B) 独立标签 — 不依附文字的语义标记

```
[伏笔:红伞]
```

| 模式 | 显示 |
|------|------|
| 显示标签 | **[伏笔: 红伞]**（带颜色标记） |
| 隐藏标签 | 完全不可见 |

冒号前是类型，冒号后是值。没有冒号时整串既是类型也是值。

#### C) 注释标签 — 写作备注

系统内置唯一特殊类型：`注`（note），用于写作过程中的自我备注。

```
他突然停下了脚步。[注:这里要铺垫转折]
```

| 模式 | 显示 |
|------|------|
| 显示标签 | 他突然停下了脚步。*[注: 这里要铺垫转折]* |
| 隐藏标签 | 他突然停下了脚步。 |

`注` 是唯一由系统识别的类型，用于区分"语义标注"和"写作备注"。其他所有标签类型均由用户自定义。

### 2.4 自定义定界符

用户在 meta 或全局设置中配置：

```json
{
  "label_delimiter": ["[", "]"]
}
```

可选示例：

| 配置 | 效果 |
|------|------|
| `["[", "]"]` | `张三[角色]` （默认） |
| `["《", "》"]` | `张三《角色》` |
| `["{", "}"]` | `张三{角色}` |
| `["<", ">"]` | `张三<角色>` |

切换定界符后，解析器自动适配。用户也可选择是否批量转换已有文档中的标签。

## 三、标签管理系统

### 3.1 用户自定义标签类型

用户在设置中管理标签类型，每个类型包含：

```json
{
  "name": "角色",
  "color": [255, 100, 100],
  "description": "故事中的人物"
}
```

- **name** — 类型名称，即定界符内写入的文字
- **color** — 该类型标签的显示颜色
- **description** — 可选说明

### 3.2 标签自动注册

当用户在正文中使用未定义过的标签类型时：

1. 自动注册为新类型
2. 从预设调色板轮选默认颜色
3. 侧边栏即时出现新分类
4. 用户可随时在设置中修改颜色和说明

### 3.3 标签类型持久化

- 文档级：`.zq` 文件的 meta 中可包含 `label_types` 字段
- 全局级：用户全局配置中可设默认标签类型

## 四、模板系统

### 4.1 模板是什么

模板 = **结构建议**，不是强制约束。模板定义：

1. 建议的标题结构（哪些章节）
2. 建议的标签类型（用户可增删改）
3. 校验规则（检查缺失的章节或标签）

### 4.2 内置模板

#### 自由写作
```json
{
  "name": "自由写作",
  "description": "无结构约束，完全自由",
  "headings": { "required": [], "optional": [] },
  "label_types": { "recommended": [] }
}
```

#### 八股文
```json
{
  "name": "八股文",
  "description": "传统八股文结构",
  "headings": {
    "required": ["破题", "承题", "起讲", "入手", "起股", "中股", "后股", "束股"],
    "optional": []
  },
  "label_types": {
    "recommended": ["对仗", "引经", "典故"]
  }
}
```

#### APA 学术论文
```json
{
  "name": "APA",
  "description": "APA 格式学术论文",
  "headings": {
    "required": ["Abstract", "Introduction", "Method", "Results", "Discussion", "References"],
    "optional": ["Appendix", "Acknowledgments"]
  },
  "label_types": {
    "recommended": ["cite", "definition", "claim", "evidence"]
  }
}
```

#### 剧本
```json
{
  "name": "剧本",
  "description": "影视剧本格式",
  "headings": { "required": [], "optional": [] },
  "label_types": {
    "recommended": ["角色", "场景", "动作", "对白"]
  }
}
```

选择模板后：
- 模板推荐的标签类型**自动预注册**到当前文档
- 用户可自由增删改
- 可随时切换或取消模板

### 4.3 自定义模板

用户创建自己的模板，保存到 `templates/` 目录，格式同上。

### 4.4 模板校验

实时校验，仅为提示，不阻止写作：

- 标题检查：哪些 required 标题缺失
- 标签检查：哪些 recommended 类型尚未使用
- 侧边栏显示完成进度百分比

## 五、功能模块

### 5.1 标签引擎 (zq-core)

- 解析标签语法 → TypedLabel（支持自定义定界符）
- 三种标签：Annotation（标注）、Metadata（独立）、Note（注释）
- 标签剥离（hide 模式）
- 标签提取（侧边栏 / 统计）

### 5.2 模板引擎 (zq-core)

- 加载模板定义
- 校验文档结构
- 计算完成度
- 管理内置 + 自定义模板

### 5.3 解析渲染 (zq-core)

- Markdown 解析
- 标签解析叠加
- 主题样式映射
- 输出 StyledSpan

### 5.4 编辑器 (zq-desktop)

- 多标签页
- 标签感知高亮
- 一键隐藏/显示标签
- 标签类型管理面板（增删改颜色）
- 自定义定界符设置
- 自动保存

### 5.5 侧边栏 (zq-desktop)

| 模式 | 内容 |
|------|------|
| 文件树 | 文件浏览器 |
| 大纲 | Markdown 标题层级，点击跳转 |
| 语义 | 用户自定义标签按类型分组，显示计数 |
| 模板 | 模板章节列表 + 完成状态 |

### 5.6 导出 (zq-desktop)

| 格式 | 处理 |
|------|------|
| .zq | 完整格式（含 meta + 标签） |
| .md | 标签剥离，保留 Markdown |
| .txt | 标签 + 格式均剥离 |
| .pdf | 渲染输出 |

### 5.7 VS Code 扩展 (zq-vscode)

- WASM 语法高亮
- 标签装饰
- 大纲视图
- 模板校验状态栏

## 六、渐进增强层级

```
Level 0 — 纯文本：直接写，不加标记
Level 1 — Markdown：加标题、粗体、列表
Level 2 — 语义标签：加 [用户自定义类型] 标注
Level 3 — 模板引导：选择模板，获得结构建议
```

每层向上兼容。Level 3 去掉标签 = Level 1，去掉格式 = Level 0。
